Tutorial: forEach y map en JavaScript

forEach y map son dos métodos de array en JavaScript que te permiten iterar sobre los elementos de un array. Aunque similares en algunos aspectos, tienen diferencias importantes en su uso y en lo que devuelven.

1. forEach

forEach es un método que ejecuta una función proporcionada una vez para cada elemento del array.

Sintaxis:

array.forEach(function(currentValue, index, arr), thisValue)

Ejemplo de uso:

            
                const numeros = [1, 2, 3, 4, 5];

                numeros.forEach(function(numero) {
                    console.log(numero * 2);
                });

                // Imprime:
                // 2
                // 4
                // 6
                // 8
                // 10

Características principales:

  • No devuelve un nuevo array.
  • Es útil cuando quieres realizar una operación o efecto secundario para cada elemento del array.
  • No puedes detener la iteración (no puedes usar break).

2. map

map crea un nuevo array con los resultados de llamar a una función proporcionada en cada elemento del array original.

Sintaxis:

let nuevoArray = array.map(function(currentValue, index, arr), thisValue)

Ejemplo de uso:

            
                const numeros = [1, 2, 3, 4, 5];

                const numerosDuplicados = numeros.map(function(numero) {
                    return numero * 2;
                });

                console.log(numerosDuplicados);
                // Imprime: [2, 4, 6, 8, 10]

Características principales:

  • Devuelve un nuevo array con la misma longitud que el original.
  • Es útil cuando quieres transformar los elementos de un array.
  • No modifica el array original.

Comparación entre forEach y map

Característica forEach map
Valor de retorno undefined Nuevo array
Modifica el array original No (pero puede modificar los elementos) No
Uso principal Ejecutar una función para cada elemento Transformar elementos y crear un nuevo array
Encadenable con otros métodos No Sí

Cuándo usar cada uno

Usa forEach cuando:

  • Quieres ejecutar código para cada elemento sin crear un nuevo array.
  • Estás realizando operaciones con efectos secundarios (como actualizar una base de datos).
  • No necesitas un nuevo array basado en el original.

Usa map cuando:

  • Quieres crear un nuevo array basado en el original.
  • Necesitas transformar cada elemento de un array de la misma manera.
  • Planeas usar el resultado en operaciones posteriores o encadenar con otros métodos de array.
